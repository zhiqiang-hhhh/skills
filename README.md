# skills

A collection of Codex skills.

## Included Skills

- `feishu-doc-reader`: Feishu/Lark wiki and docx reader with helper scripts, references, and agent config.
- `jira`: Jira REST API helpers for issue access and management.
- `doris-binary-upload`: Package and upload Doris FE/BE binaries to COS with upload metadata.

## Install

This repo installs skills into the Codex skill directory, which is usually `~/.codex/skills`.

Clone and install from GitHub:

```bash
git clone git@github.com:zhiqiang-hhhh/skills.git
cd skills
./install.sh jira feishu-doc-reader
```

List installable skills:

```bash
./install.sh --list
```

Install one skill:

```bash
./install.sh jira
./install.sh feishu-doc-reader
./install.sh doris-binary-upload
```

Install both skills:

```bash
./install.sh jira feishu-doc-reader
```

Install all skills in this repo:

```bash
./install.sh --all
```

By default the installer uses `symlink`, which is better for local development because updates in this repo take effect immediately. If you want a standalone copy instead, use `copy` mode:

```bash
./install.sh --method copy jira feishu-doc-reader
```

If a target skill already exists, replace it with `--force`:

```bash
./install.sh --force jira
```

You can also install into a custom target directory:

```bash
./install.sh --target /path/to/skills jira
```

## Skill Notes

`jira`

- Requires `python3` and the `requests` package.
- Uses these environment variables:
  `JIRA_URL`, `JIRA_USER`, `JIRA_TOKEN`
- Optional variables:
  `JIRA_AUTH_MODE`, `JIRA_API_VERSION`

`feishu-doc-reader`

- Requires `python3`.
- Uses these environment variables:
  `FEISHU_APP_ID`, `FEISHU_APP_SECRET`
- `FEISHU_TENANT_ACCESS_TOKEN` can be provided directly or generated by the bundled helper script.

## Layout

Each skill lives in its own directory and keeps its `SKILL.md` plus any supporting scripts or references beside it.
